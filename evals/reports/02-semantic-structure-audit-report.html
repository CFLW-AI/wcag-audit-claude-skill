Detailed assessment of all 28 success criteria. Failures are documented with evidence and remediation guidance.

Criterion Status Evidence & Notes

SC 1.3.1 Info and Relationships ❌ Fail
Multiple semantic failures:
  • .fake-h1, .fake-h2, .fake-h3 are styled divs, not semantic heading elements (<h1>–<h6>).
  • Navigation uses generic divs (.fake-nav) instead of <nav><ul><li>.
  • Breadcrumb navigation is a div with no <nav> wrapper or aria-label.
  • Data table missing <th> elements and scope attributes. Uses <td> for all cells.
  • Form inputs have no associated <label> elements; only placeholder text provided.
SC 1.3.2 Meaningful Sequence ❌ Fail
DOM order vs. visual order mismatch: The .reordered flex container uses CSS order property to rearrange items (order: 3, 1, 2) so visual reading order does not match DOM source order. Screen reader users will encounter items in wrong logical sequence.
SC 1.3.3 Sensory Characteristics ✅ Pass
No instructions rely solely on shape, color, or visual location. Button text reads "Send Message" (not "click the blue button"). Navigation and form controls are not described by color alone.
SC 1.4.1 Use of Color ✅ Pass
Color is not the sole means of conveying information. Links are identified as links (not by color alone); toggle switch has visual indicator (circle position); form controls not distinguished by color alone.
SC 1.4.2 Audio Control N/A
No auto-playing audio on page.
SC 1.4.3 Contrast (Minimum) ❌ Fail
Color contrast failures:
  • Form input border: #cccccc on #fefefe = 1.59:1 (fails 3:1 minimum for UI components)
  • Toggle off circle: #ffffff on #cccccc = 1.61:1 (fails 3:1 minimum)
  • Footer text: #888888 on #fefefe = 3.51:1 (fails 4.5:1 minimum for normal text)
SC 1.4.4 Resize Text ✅ Pass
No fixed-height containers clipping text. Page uses relative font sizes and flexible layout that reflows at 200% zoom without loss of content.
SC 1.4.5 Images of Text N/A
No images of text. All text rendered as actual text.
SC 1.4.10 Reflow ✅ Pass
Page reflows at 320px viewport width (400% zoom) without horizontal scrolling. Flexible layout, no fixed-width containers blocking reflow.
SC 1.4.11 Non-text Contrast ❌ Fail
UI component contrast failures:
  • Form input borders: #cccccc = 1.59:1 against #fefefe (needs 3:1)
  • Toggle switch off state: circle and border both fail 3:1 threshold
SC 1.4.12 Text Spacing ✅ Pass
Page uses flexible line-height (1.7) and margin spacing. CSS does not restrict overflow on text, allowing custom spacing adjustments to work without clipping.
SC 1.4.13 Content on Hover or Focus ✅ Pass
No hover/focus-triggered content (tooltips, dropdowns, modals). Link hover state uses standard underline, no new content appears.
SC 2.1.1 Keyboard ❌ Fail
Custom toggle lacks keyboard support: The toggle widget uses onclick only and is not keyboard accessible. No role="switch", no tabindex, no keyboard event handlers (Enter/Space). Screen reader users and keyboard-only users cannot operate this control.
SC 2.1.2 No Keyboard Trap ✅ Pass
No modal dialogs or focus traps. Keyboard focus can move freely through all elements. Tab and Shift+Tab navigate without being stuck.
SC 2.4.3 Focus Order ✅ Pass
No explicit tabindex values disrupting natural tab order. Focus order follows logical reading sequence (header nav, main content, footer).
SC 2.4.5 Multiple Ways ✅ Pass
Page provides navigation menu and links to related pages. Users can navigate via header links and breadcrumb trail.
SC 2.4.6 Headings and Labels ❌ Fail
Generic link text without sufficient context:
  • <a href="/standards">click here</a> — does not describe destination or purpose
  • <a href="/guide">Read more</a> — vague, does not indicate what "more" refers to without surrounding context
Screen reader users scanning links by themselves will not understand these links' purposes.
SC 2.4.7 Focus Visible ✅ Pass
Standard browser focus indicators (outline) are present on interactive elements. No outline: none removing focus visibility without replacement.
SC 2.4.11 Focus Not Obscured (Minimum) ✅ Pass
No sticky headers or fixed overlays obscuring focused elements. Focus indicators are fully visible at all times. No z-index layering issues.
SC 2.5.3 Label in Name ❌ Fail
Form inputs lack associated labels: Inputs use placeholder only, no <label> elements with for attribute. Accessible names are missing or rely on placeholder text, which is not a substitute for labels. Assistive technology cannot reliably announce input purposes.
SC 2.5.8 Target Size (Minimum) ✅ Pass
Interactive targets (buttons, links, form inputs) are adequately sized. Button is 34px × 24px (meets 24×24 minimum). Links and inputs meet or exceed 24px height.
SC 3.1.1 Language of Page ✅ Pass
<html lang="en"> properly declares page language as English.
SC 3.1.2 Language of Parts ❌ Fail
French text missing language attribute: The quote paragraph contains French text ("L'accessibilité n'est pas une option...") but the paragraph has no lang="fr" attribute. Screen readers will not switch to French pronunciation.
SC 4.1.2 Name, Role, Value ❌ Fail
Custom toggle missing ARIA attributes:
  • Element has no role="switch"
  • No aria-checked attribute to indicate state
  • No aria-label describing purpose
  • Not keyboard accessible (no tab index, no key handlers)
Assistive technology cannot determine the control's name, role, or current state.

Summary of Failures

9 Failures Identified:
  1. SC 1.3.1 — Info and Relationships: Semantic HTML not used (fake headings, nav divs, table missing <th>, form labels missing)
  2. SC 1.3.2 — Meaningful Sequence: CSS flexbox order property reorders visual layout without matching DOM
  3. SC 1.4.3 — Contrast (Minimum): Footer text and form borders fail minimum contrast ratios
  4. SC 1.4.11 — Non-text Contrast: Form input and toggle borders fail 3:1 UI component contrast requirement
  5. SC 2.1.1 — Keyboard: Custom toggle control not keyboard accessible
  6. SC 2.4.6 — Headings and Labels: Generic link text "click here" and "Read more" do not describe purpose
  7. SC 2.5.3 — Label in Name: Form inputs lack <label> elements; placeholder-only "labels" insufficient
  8. SC 3.1.2 — Language of Parts: French quote paragraph missing lang="fr" attribute
  9. SC 4.1.2 — Name, Role, Value: Custom toggle missing role, aria-checked, and keyboard support
