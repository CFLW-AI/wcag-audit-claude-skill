⚠️ Non-text Contrast Failures (SC 1.4.11)

Four UI component borders fail to meet the required 3.0:1 contrast ratio for non-text elements. These are subtle, low-contrast borders:

  • Button border: 1.50:1 (need 3.0:1)
  • Form input border: 1.88:1 (need 3.0:1)
  • Card border: 1.16:1 (need 3.0:1)
  • Table border: 1.24:1 (need 3.0:1)

These subtle divider lines should be brightened to meet accessibility requirements, or the borders should be replaced with alternative visual indicators (shadows, spacing) that don't rely on color contrast alone.

Recommendations

To achieve WCAG 2.2 AA compliance, address the four non-text contrast failures:

  1. Button borders: Increase #1a5276 to approximately #3b7aad or use a shadow effect instead of a thin border.
  2. Form input borders: Increase #4a4a6a to approximately #7a7a9a to reach 3:1 on #16213e background.
  3. Card borders: Increase #2a2a4a to approximately #5a5a7a or use box-shadow for visual separation.
  4. Table borders: Increase #2a2a4a to approximately #5a5a7a or use background color alternation instead of borders.

All other 24 criteria pass. The page demonstrates strong accessibility practices: semantic HTML, proper ARIA usage, keyboard navigation, high text contrast, appropriately-sized touch targets, visible focus indicators, and responsive design.

Methodology

This audit followed WCAG 2.2 Level AA conformance requirements with programmatic color contrast verification. All color pairs were extracted from CSS (including custom properties, interactive states, and contextual backgrounds) and tested using the W3C relative luminance formula. Manual review covered all 28 applicable success criteria with special attention to:

  • Semantic HTML structure and relationships (SC 1.3.1)
  • Color contrast for text and non-text elements (SC 1.4.3, SC 1.4.11)
  • Focus visibility and keyboard navigation (SC 2.4.7, SC 2.1.1)
  • Target size and focus obscuring (SC 2.5.8, SC 2.4.11 — new in WCAG 2.2)
  • Alternative text and accessible names (SC 1.1.1, SC 2.5.3)
